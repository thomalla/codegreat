LEKCJA 1: Instrukcja warunkowa IF...ELSE

Wytłumaczenie tej instrukcji nie może się obejść od odwołania do Twojego codziennego życia. Każdego dnia musisz podejmować decyzje, od momentu przebudzenia decydujesz co zjesz na śniadanie, w co się ubierzesz, czasem podejmować musisz decyzje nie tak błahe jak te związane z codziennością ale mające duży wpływ na Twoje życie np. wybór studiów, wzięcie kredytu. Komputer również podejmuje decyzje. Jest to co prawda inny proces, niż w przypadku podejmowania decyzji przez człowieka- przecież to tylko maszyna. Programista za pomocą odpowiednich narzędzi jest w stanie narzucić komputerowi podjęcie danej decyzji w danym momencie, powiedzieć jak ma się zachować jeśli zostaną spełnione odpowiednie warunki.

Jednym z tych narzędzi jest instrukcja warunkowa IF( z ang. if oznacza jeśli). Instrukcja ta mówi, że jeśli zostaną spełnione odpowiednie warunki, to komputer ma wykonać daną operację. Uwzględnia ona możliwość sytuacji, w której dane warunki nie zostaną spełnione. W przypadku takiej sytuacji, również można narzucić komputerowi wykonanie jakiś narzuconych operacji. Część instrukcji warunkowej if, po której w przypadku niespełnienia warunków komputer wykonuje operacje, rozpoczyna się od słowa else.

WARUNEK IF

Składnia warunku if

IF( warunek)

{

// operacje, które się wykonają, gdy warunek zostanie spełniony

}

Czym może być warunek? Warunkiem będzie wartość wyrażenia logicznego tj. true bądź false. Najprościej będzie przeanalizować to zagadnienie na przykładach.


PRZYKŁAD1:

#include <iostream>

using namespace std;


int main()

{

bool a(true);


if (a == true)

{

cout << "zmienna a to true" << endl;

}


}

Zauważ, że w instrukcji if sprawdzamy, czy wartość naszej zmiennej jest równa true, zwracana wartość tego wyrażenia to true lub false, jeśli true to na konsoli wyświetla się komunikat.

Równoznacznym dla tej instrukcji warunkowej będzie:

#include <iostream>

using namespace std;


int main()

{

bool a(true);


if (a)

{

cout << "zmienna a to true" << endl;

}


}

Gdyż automatycznie kompilator będzie sprawdzał czy wartość zmiennej a to true.


PRZYKŁAD 2:

#include <iostream>

using namespace std;


int main()

{

int a(10);


if (a < 20)

{

cout << "a mniejsze od 20" << endl;

}


}

W instrukcji warunkowej sprawdzany jest warunek czy wartość zmiennej a jest mniejsza od 20. Jest mniejsza, zwracana jest zatem wartość true i wyświetlony zostaje komunikat. W przypadku, gdyby wartość zmiennej a była większa od 20:

#include <iostream>

using namespace std;


int main()

{

int a(100);


if (a < 20)

{

cout << "a mniejsze od 20" << endl;

}

}

Na konsoli nie wyświetli się żaden komunikat, gdyż nie zostanie spełniony warunek w instrukcji warunkowej:

W instrukcji warunkowej mogą się znajdować bardziej skomplikowane wyrażenia:

#include <iostream>

using namespace std;


int main()

{

int a(10);

int b(12);

bool w(false);


if (a <b && b> 2 || !w )

{

cout << "zwrocono true" << endl;

}

}



Wspomnieliśmy wcześniej, że programista może narzucić wykonanie operacji komputerowi, w przypadku, gdy nie zostaną spełnione warunki, zatem w instrukcji warunkowej zostanie zwrócone false. Instrukcja if... else ma następującą postać:

IF( warunek)

{

// operacje, które się wykonają, gdy warunek zostanie spełniony

}

else

{

// operacje, które się wykonają, gdy warunek zostanie spełniony

}

ZAGNIEŻDZANIE WARUNKÓW

Musisz wiedzieć, że instrukcje warunkowe można zagnieżdżać np:

#include <iostream>

using namespace std;


int main()

{

int a(11);

if (a > 0)

{

cout << "a to liczba dodatnia" << endl;

if (a % 2 == 0)

{

cout << "a to liczba parzysta" << endl;

}

else

{

cout << "a to liczba nieparzysta" << endl;

}

}

else

{

cout << "a to liczba ujemna" << endl;

}


}

W powyższym programie gdy pierwszy warunek zostanie spełniony, sprawdzamy kolejny warunek. Wynikiem działania tego programu jest:

Na początku sprawdzany jest warunek sprawdzający czy dana liczba jest dodatnia, ponieważ liczba 11 jest dodatnią, przechodzimy do sprawdzenia kolejnego warunku. Gdyby 11 nie była liczbą dodatnią to nie przeszlibyśmy do zagnieżdżonej instrukcji warunkowej, tylko wyświetlony byłby komunikat: a to liczba ujemna. Przechodzimy do kolejnego warunku. W nim sprawdzamy, czy nasza liczba jest parzysta, ponieważ nie jest, wyświetlony zostaje komunikat z części else ( wartość warunku to false) : a to liczba nieparzysta.



INSTRUKCA ELSE IF - DODAWANIE KOLEJNYCH WARUNKÓW

Programując możemy się spotkać z sytuacją, gdy porównując daną wartość możemy uzyskać więcej niż 2 możliwości np. Jeśli liczba piłek to 1 to wykonaj dana operację, jeśli liczba piłek jest <10 to wykonaj daną operację, jeśli liczba piłek = 100 to wykonaj daną operację, w pozostałych przypadkach wykonaj inna operację. Aby utworzyć taki zestaw warunków wykorzystamy instrukcję else if ( w przeciwnym razie jeśli). Popatrz na przykład:

#include <iostream>

using namespace std;


int main()

{

int liczbaPilek(40);


if (liczbaPilek == 1)

{

cout << "Ostatnia piłka!" << endl;

}

else if (liczbaPilek < 10)

{

cout << "Zamów piłki! " << endl;

}

else if (liczbaPilek < 100)

{

cout << "Liczba piłek jest odpowiednia" << endl;

}

else if (liczbaPilek == 100)

{

cout << "Magazyn pełny" << endl;

}

else

{

cout << "Liczba piłek jest nieodpowiednia" << endl;

}

}

Mamy 40 piłek. Komputer na początku sprawdza, czy mamy 1 piłkę. Wynik jest negatywny, zatem kompilator przechodzi do pierwszej instrukcji else if. Wynik również jest negatywny, więc komputer przechodzi do kolejnej instrukcji else if, wynik jest pozytywny, zatem jest wyświetlony komunikat Liczba piłek jest odpowiednia. Program kończy swoją działalność, nie są sprawdzane pozostałe warunki. W przypadku, gdy dla każdej instrukcji else if zwrócony wynik byłby negatywny- wyświetlony zostałby komunikat Liczba piłek jest nieodpowiednia- który zostanie zwrócony zawsze, gdy nie zostaną spełnione pozostałe warunki if i else if.
